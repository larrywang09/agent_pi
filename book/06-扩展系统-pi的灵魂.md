# 第 6 章 扩展系统：pi 的灵魂

## 6.1 什么是扩展？

扩展（Extension）是 pi 最强大的能力。简单来说，**扩展是一个 TypeScript 模块，它默认导出一个函数，接收 `ExtensionAPI` 对象，通过它注册自定义行为**。

扩展可以做到的事情：

- 注册新的 LLM 工具（`registerTool`）
- 监听 Agent 生命周期事件（`on("turn_start", ...)`）
- 注册用户命令（`registerCommand`）
- 注册快捷键（`registerShortcut`）
- 注册 CLI 标志（`registerFlag`）
- 注册自定义 Provider（`registerProvider`）
- 注册自定义消息渲染器（`registerMessageRenderer`）
- 修改系统提示词（`beforeAgentStart`）
- 与用户交互（通过 `ctx.ui` 对象）
- 修改输入文本（`inputTransform`）

## 6.2 扩展发现与加载

pi 在启动时通过 `ResourceLoader` 自动发现扩展：

```typescript
// src/core/resource-loader.ts
class DefaultResourceLoader {
    async reload() {
        // 1. 扫描 .pi/extensions/ 目录
        // 2. 扫描 ~/.pi/agent/extensions/ 目录
        // 3. 扫描 --extensions 参数指定的路径
        // 4. 加载每个扩展的 index.ts（或 package.json 中的 main 字段）
        // 5. 调用扩展的默认导出函数
    }
}
```

加载路径优先级：

```typescript
// 项目级别
<project>/.pi/extensions/*/index.ts
<project>/.pi/extensions/*/package.json

// 用户全局级别
~/.pi/agent/extensions/*/index.ts
~/.pi/agent/extensions/*/package.json

// CLI 参数
--extensions /path/to/extension.ts
```

### TypeScript 运行时加载

扩展文件是 `.ts` 文件，pi 使用 `jiti`（一个 TypeScript 加载器）在运行时编译和执行它们：

```typescript
import { createJiti } from "jiti";
const jiti = createJiti(import.meta.url);
const extensionModule = await jiti.import(extensionPath);
```

### 事件处理示例

```typescript
export default function (pi: ExtensionAPI) {
    // 在用户消息发送给 LLM 之前，添加上下文
    pi.on("context", async (event, ctx) => {
        // event 包含系统提示词和消息列表
        // 可以添加额外消息到上下文
        event.messages.unshift({
            role: "user",
            content: [{ type: "text", text: "Additional context..." }],
            timestamp: Date.now(),
        });
    });

    // 在每个工具执行前做安全检查
    pi.on("tool_call", async (event, ctx) => {
        if (event.toolName === "bash") {
            // 检查命令是否危险
            if (event.args.command.includes("rm -rf /")) {
                event.block({ reason: "Dangerous command blocked" });
                return;
            }
        }
    });

    // 在 LLM 请求前注入额外的提示词
    pi.on("before_agent_start", async (event, ctx) => {
        event.actions.appendSystemPrompt("Remember to explain your reasoning.");
    });
}
```

## 6.4 ExtensionAPI 详解

`ExtensionAPI` 是扩展接收到的入口对象，包含以下主要方法：

```typescript
interface ExtensionAPI {
    // === 工具注册 ===
    registerTool(tool: ToolDefinition): void;

    // === 命令注册 ===
    registerCommand(name: string, command: SlashCommand): void;

    // === 快捷键注册 ===
    registerShortcut(keys: string[], command: string): void;

    // === CLI 标志注册 ===
    registerFlag(flag: ExtensionFlag): void;

    // === Provider 注册 ===
    registerProvider(api: string, provider: ApiProvider): void;

    // === 事件监听 ===
    on(event: "turn_start", handler: (event, ctx) => void): void;
    on(event: "turn_end", handler: (event, ctx) => void): void;
    on(event: "tool_call", handler: (event, ctx) => void): void;
    on(event: "tool_result", handler: (event, ctx) => void): void;
    on(event: "user_bash", handler: (event, ctx) => void): void;
    on(event: "message_start", handler: (event, ctx) => void): void;
    on(event: "message_update", handler: (event, ctx) => void): void;
    on(event: "message_end", handler: (event, ctx) => void): void;
    on(event: "input", handler: (event, ctx) => void): void;
    on(event: "context", handler: (event, ctx) => void): void;
    on(event: "before_agent_start", handler: (event, ctx) => void): void;
    on(event: "before_provider_request", handler: (event, ctx) => void): void;
    on(event: "session_start", handler: (event, ctx) => void): void;
    on(event: "session_shutdown", handler: (event, ctx) => void): void;
    on(event: "session_before_compact", handler: (event, ctx) => void): void;
    on(event: "session_compact", handler: (event, ctx) => void): void;
    on(event: "session_before_fork", handler: (event, ctx) => void): void;
    on(event: "session_before_switch", handler: (event, ctx) => void): void;
    on(event: "session_before_tree", handler: (event, ctx) => void): void;
    on(event: "session_tree", handler: (event, ctx) => void): void;
    on(event: "resources_discover", handler: (event, ctx) => void): void;
    on(event: "agent_start", handler: (event, ctx) => void): void;
    on(event: "agent_end", handler: (event, ctx) => void): void;

    // === 消息渲染器注册 ===
    registerMessageRenderer(renderer: MessageRenderer): void;

    // === UI 方法 ===
    setStatus(key: string, text: string | undefined): void;
    setWidget(key: string, component: ...): void;
    setFooter(factory: ...): void;
    setHeader(factory: ...): void;

    // === 事件广播 ===
    emit(eventType: string, data: unknown): void;
}
```

## 6.5 自定义工具

注册一个 LLM 调用的工具是最常见的扩展用例。

### 定义工具参数

使用 `typebox` 的 `Type.Object` 定义工具参数 Schema（自动转换为 JSON Schema 发送给 LLM）：

```typescript
import { Type } from "@earendil-works/pi-ai";

const GreetParams = Type.Object({
    name: Type.String({ description: "Name to greet" }),
    greeting: Type.Optional(
        Type.String({ description: "Custom greeting" })
    ),
});
```

### 注册工具

```typescript
export default function (pi: ExtensionAPI) {
    pi.registerTool({
        name: "greet",
        label: "Greet",
        description: "A simple greeting tool that says hello",
        parameters: GreetParams,

        async execute(toolCallId, params, signal, onUpdate, ctx) {
            const greeting = params.greeting || "Hello";
            return {
                content: [
                    { type: "text", text: `${greeting}, ${params.name}!` }
                ],
                details: {
                    greetedName: params.name,
                    usedGreeting: greeting,
                },
            };
        },

        // 可选：自定义渲染（在交互模式中工具调用的显示方式）
        renderCall(args, theme, ctx) {
            // 返回一个 Component 对象
        },

        renderResult(result, options, theme, ctx) {
            // 返回一个 Component 对象
        },
    });
}
```

### 简单工具快捷方式

pi 还提供了一个 `defineTool` 辅助函数，减少样板代码：

```typescript
import { Type } from "@earendil-works/pi-ai";
import { defineTool } from "@earendil-works/pi-coding-agent";

const reviewTool = defineTool({
    name: "review_code",
    label: "Review Code",
    description: "Review code for issues",
    parameters: Type.Object({
        filePath: Type.String(),
    }),
    execute: async (toolCallId, params, signal, onUpdate, ctx) => {
        // 工具逻辑
    },
});
```

## 6.8 ExtensionContext — 扩展能获得什么

当事件处理函数被调用时，它会收到一个 `ExtensionContext` 对象，提供丰富的上下文信息：

```typescript
interface ExtensionContext {
    // UI 交互（仅在交互模式下可用）
    ui: ExtensionUIContext;
    hasUI: boolean;

    // 当前工作目录
    cwd: string;

    // 会话管理器（只读）
    sessionManager: ReadonlySessionManager;

    // 模型注册中心
    modelRegistry: ModelRegistry;

    // 当前模型
    model: Model<any> | undefined;

    // Agent 状态
    isIdle(): boolean;
    signal: AbortSignal | undefined;
    abort(): void;
    hasPendingMessages(): boolean;

    // 会话控制
    shutdown(): void;
    getContextUsage(): ContextUsage | undefined;
    compact(options?: CompactOptions): void;
    getSystemPrompt(): string;
}
```

## 6.9 完整扩展示例分析

让我们分析 `examples/extensions/todo.ts` 这个比较完整的扩展，它展示了扩展系统的多种能力：

### 状态管理策略

Todo 扩展的状态管理有一个巧妙的设计——**状态存储在工具结果（Tool Result）的 `details` 字段中**：

```typescript
// 每次调用都被序列化到 JSONL 会话文件中
execute() {
    return {
        content: [{ type: "text", text: "..." }],
        details: {
            action: "add",
            todos: [...todos],  // 完整的 Todo 状态
            nextId: nextId,     // 下一个可用 ID
        },
    };
}
```

**为什么这样设计？** 因为每次工具执行的结果都写入 JSONL 会话文件，所以当用户恢复会话或创建分支时，可以通过重放工具结果来重建状态：

```typescript
const reconstructState = (ctx) => {
    todos = [];
    nextId = 1;
    for (const entry of ctx.sessionManager.getBranch()) {
        if (entry.type !== "message") continue;
        if (entry.message.role !== "toolResult" || entry.message.toolName !== "todo") continue;
        const details = entry.message.details;
        if (details) {
            todos = details.todos;
            nextId = details.nextId;
        }
    }
};

// 在会话恢复和分支切换时重建状态
pi.on("session_start", (_, ctx) => reconstructState(ctx));
pi.on("session_tree", (_, ctx) => reconstructState(ctx));
```

### 自定义 UI 渲染

Todo 扩展还实现了 `renderCall` 和 `renderResult` 两个方法，用于自定义工具调用的 TUI 渲染：

```typescript
pi.registerTool({
    // ...
    renderCall(args, theme, _context) {
        // 渲染工具调用：格式化为 "todo add 'xxx'"
        let text = theme.fg("toolTitle", theme.bold("todo ")) + theme.fg("muted", args.action);
        // ...
        return new Text(text, 0, 0);
    },

    renderResult(result, { expanded }, theme, _context) {
        // 渲染工具结果：漂亮的 Todo 列表
        const todoList = details.todos;
        let listText = theme.fg("muted", `${todoList.length} todo(s):`);
        for (const t of todoList) {
            const check = t.done ? theme.fg("success", "✓") : theme.fg("dim", "○");
            listText += `\n${check} ${theme.fg("accent", `#${t.id}`)} ${t.text}`;
        }
        return new Text(listText, 0, 0);
    },
});
```

## 6.10 章节小结

pi 的扩展系统是整个项目最出彩的设计之一。它的设计原则包括：

1. **最小接口**：扩展只需要导出一个接收 `ExtensionAPI` 的默认函数
2. **20+ 生命周期钩子**：覆盖从用户输入到 LLM 请求到工具执行到会话管理的全链路
3. **完整的 UI 原语**：`ExtensionUIContext` 提供了 select/confirm/input/editor/custom 等交互能力
4. **可组合性**：多个扩展可以同时运行，通过事件系统协作
5. **状态持久化**：通过工具结果的 `details` 字段实现分支安全的状态管理
6. **自定义渲染**：工具调用和结果可以自定义 TUI 渲染
7. **动态 Provider**：扩展可以注册全新的 LLM Provider

完整的扩展示例（95+ 个）位于 `packages/coding-agent/examples/extensions/` 目录，是学习 pi 扩展开发的最佳参考资料。
